Technical Approaches

How should we integrate the Fluid Engine into Dead4rat?

A

Monolithic Integration

Build directly into canvasEngine.js as a new "Generative Mode".

  • Highest performance
  • Seamless transition with other modes
  • Increases file complexity of the main engine
  • Harder to hot-swap or modularize
B

WFGL Source Plugin

Extend the new WFGL system to support "Source" plugins that generate content.

  • Highly modular
  • Can be reordered in the FX chain
  • WFGL is currently built for filters, not generators
  • More architectural overhead
C

Dedicated Sub-Engine

Create fluidEngine.js as an independent module. Blit the result to the main canvas.

  • Easiest to develop and debug
  • Keeps the codebase clean and decoupled
  • Slightly more memory overhead (extra FBOs)
  • Requires a new blit hook in the main loop
